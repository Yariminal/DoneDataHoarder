STEP 0 — SETUP & CONTEXT

DoneDataHoarder is installed as a Python CLI entry point: `ddh`

Available commands:
  ddh scan <dir>          # scans files into the DB
  ddh enrich              # extracts hashes, dates, MIME types
  ddh analyze             # AI vision + text analysis
  ddh dedup               # exact + perceptual duplicate detection
  ddh relate              # LLM semantic grouping
  ddh propose             # generates rename/move/tag proposals
  ddh review              # interactive review (or --auto-apply)
  ddh execute --force     # applies all APPROVED proposals to disk
  ddh stats               # shows DB statistics
  ddh doctor              # environment diagnostics

The database path defaults to `ddh.db` in the current working directory.

STEP 4 — CRITIQUE & ANALYSIS
================================================================================

Analyze the results and write a detailed critique covering:

4.1. ORGANIZATION QUALITY
    - Are folders well-structured and meaningfully named?
    - Are files grouped logically (by date, type, project, event)?
    - Are there any orphan files that should have been grouped?

4.2. NAMING QUALITY
    - Are renamed filenames descriptive and useful?
    - Did the AI preserve important context (dates, locations, people)?
    - Are there any mojibake/garbled characters in names?
    - Are Hebrew / non-ASCII filenames handled correctly?

4.3. DUPLICATE HANDLING
    - Were true duplicates correctly identified and removed?
    - Were any unique files mistakenly marked as duplicates?
    - Check the .ddh_trash folder — were the "keepers" the best files?

4.4. RELATE / SEMANTIC GROUPING
    - Were related files (e.g., CAD source + PDF export + backup) grouped?
    - Are there any false-positive groupings?

4.4. ERROR & EDGE CASES
    - Any files left in ERROR status? Why?
    - Any files skipped that shouldn't have been?
    - How did the system handle unusual formats or corrupt files?

4.5. OVERALL ASSESSMENT
    - Would you trust this on a real hoard of 100k+ files?
    - What would break at scale?
    - Suggest 3 concrete improvements.

================================================================================
TIPS FOR THE AI RUNNER
================================================================================

- If a command hangs (e.g., `analyze` waiting on Ollama), check that 
  Ollama is running: `ollama list` should show models. Start with 
  `ollama serve` if needed.

- If the DB is locked, another DoneDataHoarder process is using it. Kill it:
  Get-Process python | Stop-Process  # (careful — kills all Python)

- For faster iteration on the same test folder, you can re-use the DB 
  and run individual steps (e.g., just `propose` after tweaking namer rules).

- The `ddh doctor` command is useful for pre-flight checks.

- If you need to tweak configuration mid-test:
  ddh config --edit-naming
  ddh config --edit-phash
